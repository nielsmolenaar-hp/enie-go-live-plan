en-IE Go-Live — Technical Plan

What we need in place so Irish customers find us, click us, and we keep our rankings. Split into V1 (for the 3 July launch) and V2 (after launch). Tap Technical details under any item for the specifics.

Tags: ✅ live on staging · 🔧 to build / fix · ⚠️ open — verify against requirements.

V1 — for the launch

Found by Google

Google can read every page on load · ✅ Full content in the first response, so pages can rank. A page Google can't read earns nothing.

Technical detailsServer-side rendered. Essential blocks that must be in the raw HTML: Trustpilot, pricing on landing pages, product items on filter pages, images, FAQs, and funnel (configurator) attributes & values. No indexable content loaded only by JavaScript.

Internal linking works · 🔧 Google (and customers) can move through the site; no stranded pages or dead ends.

Technical detailsMenu, breadcrumb, product grids and related links are real anchors that resolve to live pages.

Clean heading structure · ⚠️ open One clear headline per page so Google understands the topic.

Technical detailsExactly one H1 per page, per the requirements, and read that way in the code. Open: currently two H1s in the source (mobile + desktop copies) — verify the current setup.

No placeholder text · 🔧 Customers never see {{shop_name}} and Google never indexes it.

Technical detailsNo `{{…}}` tokens on any page, in source, or in the JSON-LD. Needs checking across pages.

Each page is its own "original" (canonical) · ⚠️ deeper check Stops tracking/filter versions being treated as duplicates and splitting our ranking.

Technical detailsSelf-canonical, normalised (https, no-www, lowercase, no tracking params), never cross-locale; parameter URLs canonical correctly. Deeper check against the canonical requirements.

Language links correct (hreflang / x-default) · ⚠️ deeper check Irish searchers get the Irish page; no cannibalising en-GB.

Technical detailsReciprocal hreflang incl. en-ie; x-default → en-gb (kept). Deeper check of the full set against requirements.

We choose what Google indexes · 🔧 Index pages that earn traffic; keep thin/duplicate pages out.

Technical detailsPer-locale index setting; site not globally blocked. Per-page review is being integrated later.

Clean URLs + redirects · 🔧 Keep existing ranking value; no dead ends across the migration.

Technical detailsClean lowercase URLs with the country/language prefix. Redirects: the current redirect sheet works — verify it; parameter & hyphen URLs must 301 to their base URL (open — not fully in place); new Blog + Helpcenter URL structures checked after migration; removed products → 301, never 404; full list of identified 404s checked; no dead links.

Sitemap + robots.txt · ✅ / 🔧 Google discovers all ~6,754 pages; all bots welcome.

Technical detailsSitemap built on the new logic (live); the control/monitoring mechanism via the content engine is V2. Check: all URLs present vs the old sitemap, lastmod included, and regeneration working. robots.txt on the root domain: allows all search + AI bots, blocks only cart/checkout/account, declares the sitemap — open: query-parameter handling not fixed yet.

Fast & mobile-ready

Product-page speed (Core Web Vitals) · ⚠️ very important Fast pages rank and convert better.

Technical detailsVery important to check. PageSpeed / Lighthouse (mobile) on a product + category page: LCP ≤2.0s · CLS ≤0.05 · INP ≤150ms. Target — ship if close (product-page gate blocks badly slow pages).

Main image first, nothing jumps, light first load · ✅ Page feels instant and stable, lifting conversion.

Technical detailsHero/first image eager + preloaded; rest lazy; reserved space prevents layout shift; only first-screen JS loads up front.

Mobile essentials · ✅ Protects mobile rankings and looks right on a phone.

Technical detailsViewport tag, one responsive page (no separate mobile site / AMP), head + share tags (charset, language, Open Graph) — all live. Full mobile optimisation is V2.

Trust & rich results

Product pages show price, stock & breadcrumb in Google · ⚠️ deeper check Rich results lift click-through in the Irish results.

Technical detailsProduct + Offer + Breadcrumb structured data. Deeper check of all fields against the schema requirements.

Remove product + category star ratings · 🔧 We have no product-specific reviews, so a star rating would be misleading/non-compliant.

Technical detailsAggregateRating must be removed from product AND category pages as part of the schema improvements. Company-level rating stays.

Category pages show their product list & breadcrumb · 🔧 Helps category terms rank — the biggest Irish demand pool.

Technical detailsCategory page sends a product list + breadcrumb (modern schema). Known: still on the old CollectionPage format — fix right after.

Breadcrumb structure · 🔧 Shows customers and Google where a page sits, and powers the breadcrumb rich result.

Technical detailsVisible breadcrumb trail + matching BreadcrumbList schema on every page. Known: not working for category pages — fix.

Company Trustpilot rating shows in Google and on the page · ✅ Trust for a brand new to Irish customers.

Technical detailsCompany-level rating in structured data + server-rendered on first paint; hides cleanly if unavailable.
